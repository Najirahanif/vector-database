There are **two main types of vectors** used in modern search systems, and a third category that combines them.

| Vector Type                 | Description                                                         | Example Models                                                               | Best For                              |
| --------------------------- | ------------------------------------------------------------------- | ---------------------------------------------------------------------------- | ------------------------------------- |
| **Dense Vectors**           | Every dimension usually has a value. Captures semantic meaning.     | OpenAI `text-embedding-3-small`, Voyage AI, Sentence Transformers, ImageBind | Semantic search, recommendations, RAG |
| **Sparse Vectors**          | Most dimensions are zero. Represents keywords and their importance. | BM25, SPLADE, uniCOIL                                                        | Keyword search, exact matching        |
| **Hybrid (Dense + Sparse)** | Uses both dense and sparse vectors together.                        | Qdrant Hybrid Search, Weaviate Hybrid Search                                 | Production search systems             |

---

## 1. Dense Vectors

A dense vector looks like this:

```text
[0.24, -0.81, 0.15, 0.93, -0.44]
```

Notice that **almost every position contains a value**.

Example:

```text
Sentence:
"The train was robbed."
```

Embedding:

```text
[0.34, 0.18, -0.72, 0.91, ...]
```

Characteristics:

* Captures semantic meaning.
* Similar meanings produce similar vectors.
* Generated by neural network embedding models.

Examples of embedding models:

* OpenAI Embeddings
* Sentence Transformers
* Voyage AI
* Cohere Embed
* ImageBind
* CLIP

---

## 2. Sparse Vectors

A sparse vector has **mostly zeros**.

Conceptually:

```text
Dimension:

1     2     3     4     5     6

Value:

0     0     3.2   0     0     1.5
```

Or represented as only the non-zero entries:

```text
{
  3: 3.2,
  6: 1.5
}
```

Only a few dimensions are stored.

Characteristics:

* Represents keywords rather than semantic meaning.
* Very memory efficient.
* Excellent for exact keyword matching.

Popular sparse models:

* BM25
* SPLADE
* uniCOIL

---

## 3. Hybrid Vectors

Modern systems often use **both**.

Query:

```text
best train robbery movie
```

The search runs:

```text
Dense Vector
      +
Sparse Vector
      ↓
Rank Fusion
      ↓
Final Results
```

This combines semantic understanding with keyword matching.

## Other ways vectors are categorized

### By data type

| Type    | Description                                    |
| ------- | ---------------------------------------------- |
| Float32 | Most common for embeddings.                    |
| Float16 | Smaller memory usage, slightly less precision. |
| Int8    | Used with scalar quantization.                 |
| Binary  | 1-bit representation for binary quantization.  |

---

### By modality

| Type               | Example                                                             |
| ------------------ | ------------------------------------------------------------------- |
| Text Vectors       | Text embeddings                                                     |
| Image Vectors      | Image embeddings (e.g., CLIP, ImageBind)                            |
| Audio Vectors      | Audio embeddings                                                    |
| Video Vectors      | Video embeddings                                                    |
| Multimodal Vectors | One embedding space for text, images, audio, etc. (e.g., ImageBind) |

## In your projects

### MongoDB Movies

```text
Movie Plot
      ↓
OpenAI Embedding
      ↓
Dense Vector (1536 dimensions)
      ↓
MongoDB Vector Search
```

You are using **dense vectors**.

### Qdrant Image Search

```text
Image
      ↓
ImageBind
      ↓
Dense Vector (512 or 1024 dimensions, depending on the model)
      ↓
Qdrant
```

Again, you're using **dense vectors**.
